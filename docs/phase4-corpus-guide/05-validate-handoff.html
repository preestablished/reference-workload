Step 5 - Validate And Handoff

Assemble manifest, run validators, and publish only sanitized evidence.

Previous: Labels And Trace Overview

Goal

The private bundle is complete only when the bundle checker passes and the public fulfillment note can be updated without exposing private data.

Create Manifest

Write manifest.json in the private bundle. It must include the reference-workload commit, WorkloadImage identity or ref, operator metadata policy, feature-map hash, scoring-program hash, layout hash, capture count, framebuffer format, storage ref, access requirement, retention, compression format, max expected size, and clean-room provenance note.

Run Source Tests

cargo fmt --all -- --check
cargo test --locked -p refwork-featuremap
cargo test --locked -p refwork-verify phase4 -- --nocapture
cargo test --locked -p xtask
git diff --check

Run Private Validators

cargo run --locked -p refwork-featuremap -- validate \
  <private-bundle-dir>/feature-map.yaml \
  --scoring <private-bundle-dir>/scoring-program.yaml

cargo run --locked -p refwork-verify -- phase4-checksum-manifest \
  --bundle <private-bundle-dir> \
  --out <private-bundle-dir>/validation/checksum-manifest.json

cargo run --locked -p refwork-verify -- phase4-bundle-check \
  --bundle <private-bundle-dir> \
  --report <private-bundle-dir>/validation/phase4-bundle-check.json

Redaction Scan Public Notes

Before editing public request notes, scan the intended public text. Add a forbid-file with operator-known private literals when available. 

cargo run --locked -p refwork-verify -- redaction-scan \
  --input <public-fulfillment-note> \
  --report <private-bundle-dir>/validation/redaction-scan-report.json \
  --forbid-file <private-redaction-literals.txt>

Public Handoff May Include

  • Opaque private bundle artifact id or registry ref.
  • Role-based access requirement.
  • Retention expectation.
  • Compression format and maximum expected size.
  • Top-level hashes and validation report hashes.
  • Capture count and framebuffer metadata summary.
  • K=32 score-plan statement and trajectory coverage summary.
  • Redacted command templates.

Public Handoff Must Not Include

  • Exact ROM filename or path.
  • Exact private root path.
  • Private capture ids.
  • Decoded real feature vectors.
  • Raw WRAM, framebuffer bytes, screenshots, save RAM, or ROM bytes.
  • Operator labels or trajectory JSONL.
  • Secret retrieval commands or token-owner details.

Done Criteria

  • phase4-bundle-check passes.
  • Capture count is at least 1,000.
  • Framebuffer metadata exists for the primary corpus.
  • Feature-map/scoring validation passes.
  • Trace report passes.
  • Checksum manifest exists.
  • Redaction scan passes against the public note.
  • The fulfillment note is updated with sanitized evidence only.
