Step 1 - Private Intake

Confirm the selected ROM and prepare the lab-private workspace.

Overview Next: Map And Layout

The current workspace already passed private intake after the ROM directory was reduced to exactly one regular file. This page documents how to repeat or verify that state.

Goal

The intake step selects exactly one operator-approved ROM and writes exact ROM metadata only inside the private root. It also creates the private directory skeleton used by later steps.

Verify The Single-ROM Guard

rom_dir="$HOME/ROMs/SNES"
file_count=$(find "$rom_dir" -maxdepth 1 -type f | wc -l | tr -d ' ')
symlink_count=$(find "$rom_dir" -maxdepth 1 -type l | wc -l | tr -d ' ')
dir_count=$(find "$rom_dir" -mindepth 1 -maxdepth 1 -type d | wc -l | tr -d ' ')
printf 'file_count=%s symlink_count=%s dir_count=%s\n' \
  "$file_count" "$symlink_count" "$dir_count"

Continue only when file_count=1 and symlink_count=0. Do not print the file name in shared logs.

Run Private Intake

run_id=$(date -u +%Y%m%dT%H%M%SZ)
private_root="$HOME/.agents/private/reference-workload/phase4-scorer-golden-$run_id"

cargo run --locked -p refwork-verify -- phase4-private-intake \
  --rom-dir "$HOME/ROMs/SNES" \
  --private-root "$private_root" \
  --operator-approved \
  --operator-metadata-policy "operator ROM metadata available only inside private bundle"

Expected Result

  • Command exits successfully.
  • rom-metadata.json exists in the private root.
  • PRIVATE-RUNBOOK.md exists in the private root.
  • phase3-scorer-corpus/ exists but is still empty.

Do Not Proceed If

  • The ROM directory contains zero files, multiple files, or symlinks.
  • The private root is inside the source checkout.
  • Operator approval is missing or unclear.
