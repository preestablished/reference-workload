Phase 4 Scorer Corpus Guide

A step-by-step guide for producing the private phase3-scorer-corpus bundle and validating it for handoff.

1. Private Intake 2. Map And Layout 3. Capture Export 4. Labels And Trace 5. Validate And Handoff

Privacy Rule

Do not commit or paste ROM bytes, exact ROM filenames, exact private paths, raw WRAM, framebuffer bytes, screenshots, decoded real feature vectors, private capture ids, trajectory JSONL, operator labels, or retrieval commands into source files, public notes, tickets, or chat. Keep those details only in the lab-private root.

What You Are Building

The final private bundle is a directory named phase3-scorer-corpus. It must contain enough evidence for state-scorer to run its Phase 4 real-capture gate without asking reference-workload for extra context. 

phase3-scorer-corpus/
  manifest.json
  workload-image.yaml or workload-image-ref.txt
  feature-map.yaml
  scoring-program.yaml
  layout.json
  captures/index.jsonl
  dedup-groups.jsonl
  score-plan.json
  trajectory/first-boss.jsonl
  validation/

Current State

  • The selected ROM has passed the single-ROM private intake guard.
  • The private skeleton exists.
  • The corpus files still need to be produced and validated.
  • Reference-workload provides the real worker capture exporter, artifact verifier, context exporter, fallback checker, and recursive freeze verifier.

Workflow Summary

  1. Confirm private intake and private root setup.
  2. Create or place private feature map, scoring program, and image evidence.
  3. Generate layout.json.
  4. Run a real in-VM capture/export job to create captures/index.jsonl.
  5. Create dedup labels, operator labels, score plan, and trajectory report.
  6. Assemble manifest and validation reports.
  7. Run bundle validation and redaction scan before public handoff.

Placeholders

Commands in this guide use placeholders such as <private-root> and <capture-id>. Replace them only in your terminal or private runbook. Do not replace them in public docs.
