Step 4 - Labels, Score Plan, And Trace

Create consumer-aid files from the captured corpus.

Previous: Capture Export Overview Next: Validate And Handoff

Goal

Add labels and trajectory evidence that prove the corpus contains useful states for state-scorer: same-canonical examples, distinct-stable examples, first-boss coverage, goal-positive fixtures, and goal-negative fixtures.

Create Dedup Labels

Write dedup-groups.jsonl. Use capture ids only inside the private bundle. 

{"schema_version":1,"group_id":"dedup-001","expected_relation":"same_canonical_state","capture_ids":["<capture-id>","<capture-id>"],"changed_features":["<volatile-feature>"],"notes":"volatile-only change"}
{"schema_version":1,"group_id":"dedup-002","expected_relation":"distinct_stable_state","capture_ids":["<capture-id>","<capture-id>"],"changed_features":["<stable-feature>"],"notes":"stable feature changed"}

Create Operator Label File

The trace command expects labels with capture ids and truth values. Keep this file private. 

schema_version: 1
kind: phase4-trace-labels
labels:
  - capture_id: <capture-id>
    expected_highest_stage: <stage-name>
    prune: false
    goal: false
    first_boss_coverage: true
  - capture_id: <capture-id>
    expected_highest_stage: <stage-name>
    prune: false
    goal: true
    first_boss_coverage: false

Generate Score Plan

Pick one or more private capture ids for first-boss, goal-positive, and goal-negative evidence. The command will write deterministic K=32 batches. 

cargo run --locked -p refwork-verify -- phase4-score-plan \
  --captures <private-bundle-dir>/captures/index.jsonl \
  --out <private-bundle-dir>/score-plan.json \
  --first-boss <capture-id> \
  --goal-positive <capture-id> \
  --goal-negative <capture-id>

Emit Trajectory

mkdir -p <private-bundle-dir>/trajectory
mkdir -p <private-bundle-dir>/validation

cargo run --locked -p refwork-verify -- trace \
  --captures <private-bundle-dir>/captures/index.jsonl \
  --map <private-bundle-dir>/feature-map.yaml \
  --scoring <private-bundle-dir>/scoring-program.yaml \
  --labels <private-labels.yaml> \
  --out <private-bundle-dir>/trajectory/first-boss.jsonl \
  --report <private-bundle-dir>/validation/trace-report.json

Acceptance Checklist

  • dedup-groups.jsonl has same-canonical examples.
  • dedup-groups.jsonl has distinct-stable examples.
  • score-plan.json has K=32 batches and fixed client ids.
  • trajectory/first-boss.jsonl exists.
  • validation/trace-report.json exists and reports pass.
  • At least one first-boss, goal-positive, and goal-negative capture is labeled.
