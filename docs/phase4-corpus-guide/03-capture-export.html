Step 3 - Capture Export

Produce the real in-VM capture corpus.

Previous: Map And Layout Overview Next: Labels And Trace

Run this command only after operator approval, private intake, and worker provenance checks. All output is private game-derived data and must remain outside source checkouts.

Goal

Create captures/index.jsonl with at least 1,000 real, frame-coherent in-VM captures from the selected ROM. Each capture must pair WRAM-derived feature bytes and framebuffer metadata from the same frame boundary.

Required Output Directory

<private-bundle-dir>/captures/index.jsonl
<private-bundle-dir>/artifacts/feature-bytes/...
<private-bundle-dir>/artifacts/framebuffer/...

The exact artifact layout is flexible if every row uses private opaque refs and hashes. Do not inline raw bytes in index.jsonl.

Supported Export Command

cargo run --locked -p refwork-verify -- phase4-capture-export \
  --endpoint "$WORKER_ENDPOINT" --snapshot "$READY_SNAPSHOT_BLAKE3" \
  --padlog "$PRIVATE_TRAJECTORY_PADLOG" --map "$PRIVATE_FEATURE_MAP" \
  --layout "$PRIVATE_LAYOUT" --bundle "$PRIVATE_BUNDLE" \
  --count 1000 --cadence 1 --hard-icount-cap "$BOUNDED_RUN_CAP" \
  --source-ref "$OPAQUE_SOURCE_REF" --production

The exporter restores once, schedules the approved padlog, and uses sequential frame-budget Run calls with capture enabled. Feature bytes and the compressed framebuffer therefore come from the same pause boundary. A positive instruction cap is mandatory.

Capture Row Contract

{
  "schema_version": 1,
  "capture_id": "<private-capture-id>",
  "node_ref": "<source-node-or-state-ref>",
  "capture_source": "<tool-or-rpc-name>",
  "frame_index": 120,
  "layout_hash": "blake3:<hex>",
  "feature_bytes": {
    "ref": "artifact:<opaque-private-ref>",
    "len": 4096,
    "blake3": "blake3:<feature-artifact-hex>"
  },
  "decoded_order": ["room_id", "area_id", "player_x"],
  "decoded_values": [1, 0, 144],
  "framebuffer": {
    "ref": "artifact:<opaque-private-ref>",
    "len": 65536,
    "encoding": "fb_lz4",
    "width": 256,
    "height": 224,
    "stride": 1024,
    "pixel_format": "xrgb8888",
    "uncompressed_len": 229376,
    "blake3": "blake3:<compressed-artifact-hex>",
    "uncompressed_blake3": "blake3:<decoded-pixel-hex>"
  }
}

Exporter Requirements

  • Run the selected ROM through the approved in-VM capture path.
  • Export at least 1,000 captures.
  • Use the exact layout.json.blake3 value in every row.
  • Pack feature bytes according to layout.json.ranges.
  • Set feature_bytes.len equal to layout.json.total_len.
  • Decode values in exactly the same order as feature-map.yaml.
  • Record framebuffer metadata for the primary corpus.
  • Write private artifact refs and hashes, not inline byte arrays.

Durability And Resume

Each artifact is written to a new temporary file, synced, and renamed before its JSONL row is appended and synced. On restart, the exporter rejects duplicate ids or changed layout/source provenance, restores the same snapshot, fast-forwards to the existing row count, and appends. It never truncates or overwrites an existing corpus.

Artifact And Freeze Verification

cargo run --locked -p refwork-verify -- phase4-artifact-check \
  --bundle "$PRIVATE_BUNDLE" --report "$EXTERNAL_ARTIFACT_REPORT"
cargo run --locked -p refwork-verify -- phase4-checksum-manifest \
  --bundle "$PRIVATE_BUNDLE" --set-payload-root
cargo run --locked -p refwork-verify -- phase4-checksum-manifest \
  --bundle "$PRIVATE_BUNDLE" --out "$EXTERNAL_FREEZE_MANIFEST"
cargo run --locked -p refwork-verify -- phase4-checksum-manifest \
  --verify "$EXTERNAL_FREEZE_MANIFEST" --bundle "$RETRIEVED_BUNDLE"

framebuffer.blake3 always hashes stored LZ4 bytes; uncompressed_blake3 hashes the 229,376 decoded XRGB8888 bytes. The artifact checker rejects absolute/traversing refs, symlinks, duplicate refs, corruption, geometry drift, and either hash mismatch. The external freeze manifest recursively covers the finalized bundle; verification is read-only and rejects missing, extra, or changed files.

Derive A Context Fixture

cargo run --locked -p refwork-verify -- phase4-context-export \
  --corpus "$FROZEN_CORPUS" --out "$PRIVATE_CONTEXT_DIR" \
  --capture-id "$SELECTED_PRIVATE_CAPTURE_ID" \
  --context-artifact-id "$OPAQUE_CONTEXT_ID" \
  --access-requirement "$APPROVED_ACCESS_ROLE" --retention "$RETENTION" \
  --pad-table-hash "$CONSOLE16_TABLE_HASH" \
  [--recent-input "$APPROVED_RECENT_INPUT_PADLOG"]
cargo run --locked -p refwork-verify -- phase4-context-check \
  --bundle "$PRIVATE_CONTEXT_DIR" \
  --report "$PRIVATE_CONTEXT_DIR/validation/phase4-context-check.json"

Context export first verifies the source artifacts, then copies only selected artifacts and preserves decoded order, values, and provenance. Omit --recent-input when the operator did not approve a retained tail; the manifest records it as unavailable.

Useful Local Validation While Exporting

After writing a draft captures/index.jsonl, run the bundle checker as soon as the other required files exist. It will catch layout length drift, missing framebuffer metadata, decoded-order drift, and capture count problems. 

cargo run --locked -p refwork-verify -- phase4-bundle-check \
  --bundle <private-bundle-dir> \
  --report <private-bundle-dir>/validation/phase4-bundle-check.json

Stop Conditions

  • Fewer than 1,000 real captures are available.
  • Framebuffer metadata is missing from the primary corpus.
  • Captured WRAM and framebuffer are not from the same frame boundary.
  • The exporter decodes features in an order different from the feature map.
