Step 3 - Capture Export

Produce the real in-VM capture corpus.

Previous: Map And Layout Overview Next: Labels And Trace

This is the main missing producer step. Reference-workload currently has validators and helper generators for the corpus contract. The real capture exporter must come from the Phase 3 in-VM capture path or be added as a private producer that emits the shape below.

Goal

Create captures/index.jsonl with at least 1,000 real, frame-coherent in-VM captures from the selected ROM. Each capture must pair WRAM-derived feature bytes and framebuffer metadata from the same frame boundary.

Required Output Directory

<private-bundle-dir>/captures/index.jsonl
<private-bundle-dir>/artifacts/feature-bytes/...
<private-bundle-dir>/artifacts/framebuffer/...

The exact artifact layout is flexible if every row uses private opaque refs and hashes. Do not inline raw bytes in index.jsonl.

Capture Row Contract

{
  "schema_version": 1,
  "capture_id": "<private-capture-id>",
  "node_ref": "<source-node-or-state-ref>",
  "capture_source": "<tool-or-rpc-name>",
  "frame_index": 120,
  "layout_hash": "blake3:<hex>",
  "feature_bytes": {
    "ref": "artifact:<opaque-private-ref>",
    "len": 4096,
    "blake3": "blake3:<hex>"
  },
  "decoded_order": ["room_id", "area_id", "player_x"],
  "decoded_values": [1, 0, 144],
  "framebuffer": {
    "ref": "artifact:<opaque-private-ref>",
    "encoding": "fb_lz4",
    "width": 256,
    "height": 224,
    "stride": 1024,
    "pixel_format": "xrgb8888",
    "uncompressed_len": 229376,
    "blake3": "blake3:<hex>"
  }
}

Exporter Requirements

  • Run the selected ROM through the approved in-VM capture path.
  • Export at least 1,000 captures.
  • Use the exact layout.json.blake3 value in every row.
  • Pack feature bytes according to layout.json.ranges.
  • Set feature_bytes.len equal to layout.json.total_len.
  • Decode values in exactly the same order as feature-map.yaml.
  • Record framebuffer metadata for the primary corpus.
  • Write private artifact refs and hashes, not inline byte arrays.

Useful Local Validation While Exporting

After writing a draft captures/index.jsonl, run the bundle checker as soon as the other required files exist. It will catch layout length drift, missing framebuffer metadata, decoded-order drift, and capture count problems. 

cargo run --locked -p refwork-verify -- phase4-bundle-check \
  --bundle <private-bundle-dir> \
  --report <private-bundle-dir>/validation/phase4-bundle-check.json

Stop Conditions

  • Fewer than 1,000 real captures are available.
  • Framebuffer metadata is missing from the primary corpus.
  • Captured WRAM and framebuffer are not from the same frame boundary.
  • The exporter decodes features in an order different from the feature map.
