# Image Handoff Inputs

This directory contains the static inputs for the package-04 image handoff.
`xtask image` will consume these files in the next package to build `dist/`.

- `boot.toml` is a guest-sdk-owned schema instance. The guest-sdk owns the
  `boot.toml` schema and the detguest-agent behavior that executes the
  `refwork-harness` autostart unit.
- `expected-regions.toml` is guest-sdk-owned handoff data for READY gating. It
  names `wram`, `framebuffer`, and `meta` with byte sizes and per-region
  `layout_version` values.
- `harness.toml` is reference-workload-owned harness configuration. It records
  the fd-3 protocol version, game-image device path, and optional region policy.
- `kernel.lock`, `builder.lock`, and `guest-sdk.lock` are package-04 input pins.
  Their placeholder values are resolved by the image build package before any
  distributable artifact is produced.

No game ROM, SRAM, framebuffer golden, or game-derived bytes belong in this
directory. The operator game image is attached separately by the hypervisor as
the read-only `game-image` block device.

Region `layout_version` belongs to `boot.toml` and `expected-regions.toml`,
which are guest-sdk handoff files. Do not add region `layout_version` fields to
`workload-image.yaml`; the WorkloadImage manifest is generated by `xtask image`
from API.md section 4 and owns only region names and sizes.
