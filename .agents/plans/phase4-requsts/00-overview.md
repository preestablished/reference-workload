## Real Capture Evidence Floor

Use this exact name in the package files. The floor is met only when a durable
lab-private report proves all of the following:

- a WorkloadImage manifest hash/opaque id with the validation stamp required by
  reference-workload;
- real in-VM frame-boundary captures from the Phase 3 workload path, including
  `wram` and `framebuffer`;
- region names, sizes, layout versions, and framebuffer format metadata;
- a real feature map exercised against that region layout with a passing
  `refwork-verify map-check` report or successor validation report;
- operator-approved game or ROM revision metadata, or a statement that the
  metadata is available only inside the private bundle;
- capture/export tool command, commit or version, and report hash;
- a recent pad tail only if available; if unavailable, record why.

Until this floor is met, packages 03 and 04 may define schemas and synthetic
tool tests, but they must not claim live Phase 4 fulfillment.

## Standing Constraints

- Do not commit ROMs, raw captures, framebuffer images, screenshots, save RAM,
  game-derived byte dumps, decoded feature vectors from real captures,
  trajectory JSONL, operator labels, recent padlog tails, private capture ids,
  exact private paths, or unapproved proprietary identifiers unless the artifact
  is synthetic or separately approved for publication.
- Markdown in public or semi-public request locations may contain only opaque
  artifact ids, hashes, counts, sanitized metadata, redacted command templates,
  and role-based access requirements. Put direct paths, token-owner names, exact
  private command arguments, and retrieval steps in lab-private storage unless
  the request owner explicitly marks the request location private enough for

  references, public test ROM suites, and operator-supplied lab artifacts only.

  `scoring-program.yaml` inside the private bundle and leave repo placeholders
